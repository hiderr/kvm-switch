# KVM Switch

Мгновенное переключение клавиатуры и трекпада между двумя Mac по локальной сети
(software KVM). Девайсы остаются спарены по Bluetooth с одним Mac — Bluetooth не
трогается вообще. Ввод форвардится по TCP, переключение по хоткею мгновенное.

**Симметрично:** на обоих Mac стоит одно и то же приложение, каждый может и отдавать
ввод, и принимать. Всё управление — из иконки в меню-баре, без терминала.

## Почему не Bluetooth-свитчер

Apple Magic Trackpad/Keyboard хранят бонд только с одним хостом — любой свитчер
переспаривает девайс на каждом переключении (10–15 с и отвалы). KVM это обходит.

## Сборка (на обоих Mac)

```sh
./build.sh        # нужен Xcode CLT: xcode-select --install
open "KVM Switch.app"
```

## Права (System Settings → Privacy & Security)

Перетащи `KVM Switch.app` и включи:

- **Accessibility** — обязательно (и для перехвата, и для воспроизведения ввода)
- **Input Monitoring** — обязательно (перехват клавиатуры)
- **Локальная сеть** — macOS спросит при первом запуске (для авто-поиска)

Ad-hoc подпись (в `build.sh`) фиксирует identity, права не слетают при пересборке.

## Иконка в меню-баре

- 🔴 связи нет
- 🟢 связь есть, ввод на этом Mac
- 🔵 связь есть, ввод уходит на второй Mac

Клик по иконке показывает: имя/IP этого Mac, состояние исходящей связи (→) и
входящей (←), порт, хоткей.

## Настройка (всё из приложения)

Иконка → **Настройки…**:

- **Второй Mac** — выбор из найденных в сети (Bonjour) либо вручную IP/host + порт
- **Хоткей** — нажми кнопку и набери свою комбинацию (по умолчанию `⌃⌥⌘S`)
- **Край экрана** — тумблер + направление (где стоит второй Mac). Курсор уходит на
  второй Mac, когда упрёшься в крайнее ребро ВСЕХ твоих экранов (внутренние стыки
  мониторов не триггерят — между своими экранами ходишь свободно). Возврат — упереться
  в обратный край на втором Mac. Нужен push (~120 px), случайно не сработает.
- **Автозапуск** — тумблер «запускать при логине»

Выбрать второй Mac можно и быстро: иконка → **Второй Mac** → клик по имени.

## Как пользоваться

1. Запусти приложение на обоих Mac, выдай права.
2. На каждом в «Настройках» выбери второй как peer (или кликни в подменю «Второй Mac»).
3. Иконка станет 🟢 — связь есть.
4. Жми хоткей (`⌃⌥⌘S`) — ввод мгновенно уходит на второй Mac (🔵), ещё раз — назад (🟢).
   Можно и через иконку → «Переключить ввод».

Связь рвётся в режиме 🔵 → ввод сам возвращается на этот Mac (бип). Не залипнешь.

## Надёжность

- Прямой TCP, `noDelay`, keepalive, авто-reconnect.
- Авто-fallback в local при обрыве.
- Инжектируемые события помечаются — в симметрии нет петли «принял → отправил назад».
- Совет: дай Mac'ам статичные/зарезервированные IP (или полагайся на Bonjour-имена).

## Ограничения

Форвардятся: курсор, клики, drag, two-finger scroll, вся клавиатура.
НЕ форвардятся мультитач-жесты (свайпы Spaces, Mission Control, pinch) — на втором
Mac трекпад как мышь + скролл. Ограничение любого software-KVM.
